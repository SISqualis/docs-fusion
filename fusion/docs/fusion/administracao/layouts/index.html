Layouts

Os Layouts de Integração são um componente essencial no Fusion para garantir a correta estruturação e padronização dos dados trocados com sistemas externos. Eles definem o formato e os campos que serão utilizados, facilitando a interoperabilidade e o monitoramento das informações.

O Papel dos Layouts

Um layout de integração funciona como um "molde" ou "esquema" para os dados. Ele especifica quais informações são esperadas, em que formato e como elas se organizam. Utilizar layouts bem definidos é crucial para o sucesso das integrações, assegurando que os dados sejam interpretados corretamente pelos sistemas envolvidos.

Configurando Layouts no Fusion​

A configuração de layouts no Fusion permite uma adaptação precisa às necessidades de cada integração, suportando diversos padrões e oferecendo flexibilidade na definição dos campos.

  • Tipos de Layouts Suportados
  • Processo de Configuração Detalhada

O Fusion oferece suporte a uma variedade de tipos de layouts consagrados na indústria, especialmente no setor de saúde, para promover a interoperabilidade:

  • FHIR (Fast Healthcare Interoperability Resources): Um padrão moderno projetado para permitir a troca de informações de saúde eletrônicas de forma rápida e eficiente. Destaca-se pela sua flexibilidade e uso de tecnologias web.

  • HL7 (Health Level Seven): Um conjunto de padrões amplamente utilizado para a troca, integração, compartilhamento e recuperação de informações eletrônicas de saúde.

  • JSON (JavaScript Object Notation): Um formato leve de intercâmbio de dados, fácil para humanos lerem e escreverem, e simples para máquinas analisarem e gerarem. Amplamente utilizado em APIs web.

  • OpenEHR: Um padrão aberto para o Registro Eletrônico de Saúde (RES), focado na especificação de modelos de informação clínica robustos e longitudinais.

Padronização

O uso desses padrões facilita a comunicação entre diferentes sistemas de informação em saúde, reduzindo a complexidade e os custos de integração.

A configuração detalhada da estrutura interna de um layout no Fusion envolve várias etapas importantes:

  • Seleção da Versão: Para padrões como HL7 ou FHIR, é possível escolher a versão específica do layout que será utilizada, garantindo compatibilidade.

  • Definição de Campos (Nome do Campo): Você pode criar e nomear os campos que farão parte da estrutura do layout, especificando os dados que serão transportados.

  • Associação a Classes: Cada campo ou conjunto de campos pode ser associado a uma "classe". Essas classes são frequentemente definidas e customizadas conforme as particularidades e os requisitos do projeto do cliente, permitindo uma modelagem de dados precisa.

Atenção aos Detalhes

A correta definição da estrutura do layout, incluindo a escolha do tipo, versão, campos e classes, é fundamental. Erros ou omissões nesta etapa podem levar a falhas na comunicação e interpretação incorreta dos dados entre os sistemas integrados.

Dominar a configuração dos layouts de integração no Fusion é um passo chave para construir um ecossistema de informações de saúde interoperável e eficiente.
