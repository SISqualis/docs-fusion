# Servidores de Banco de Dados

A capacidade do Fusion de se conectar e interagir com diversas fontes de dados é crucial para sua funcionalidade, especialmente em ambientes hospitalares complexos. A seção de configuração de Servidores de Banco de Dados permite o cadastramento e gerenciamento das conexões com os bancos de dados utilizados pelo cliente e por sistemas integrados.

<Admonition type="info" title="Objetivo da Configuração" icon={<Database />}>
  <p>
    Nesta área do sistema, são registradas as informações essenciais sobre os bancos de dados com os quais o Fusion irá interagir. Isso inclui "Informações do tipo de banco de dados, qual o sistema de gestão hospitalar que o cliente utiliza, assim como o cadastramento dos dados de conexão do DB". Uma configuração precisa é vital para garantir a correta comunicação e troca de dados.
  </p>
</Admonition>

## Detalhes da Configuração da Conexão

Para estabelecer uma conexão bem-sucedida, diversos parâmetros precisam ser informados:

* **Tipo de Banco de Dados:** Especificar qual sistema de gerenciamento de banco de dados (SGBD) está sendo utilizado (ex: SQL Server, Oracle, PostgreSQL, MySQL, etc.).
* **Sistema de Gestão Hospitalar (SGH):** Identificar o sistema de gestão hospitalar (HIS - Hospital Information System) que o cliente utiliza, pois muitos dados relevantes para o Fusion podem residir no banco de dados deste sistema.
* **Dados de Conexão do DB:** Cadastrar as informações técnicas necessárias para estabelecer a conexão, tais como:
    * Endereço do servidor (host)
    * Nome da instância ou do banco de dados
    * Porta de comunicação
    * Credenciais de acesso (usuário e senha), quando aplicável e de forma segura.

A correta configuração dos servidores de banco de dados no Fusion é um pré-requisito para a integração de dados e para o funcionamento adequado de diversas funcionalidades que dependem do acesso a informações externas ou legadas.
