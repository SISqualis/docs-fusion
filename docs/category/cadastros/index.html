Cadastros

Alérgicos

O módulo de Alérgicos no Fusion permite o registro e manutenção de substâncias ou agentes que provocam reações alérgicas em pacientes. Este cadastro é essencial para garantir a segurança clínica durante atendimentos, prescrições e integrações com outros sistemas.

Categorias de Alergia

O cadastro de Categorias de Alergia permite classificar os alérgenos registrados no sistema Fusion. Essa classificação auxilia a organização das informações clínicas, a padronização de dados e a integração entre sistemas distintos.

Acomodação

O módulo de Acomodações permite que o sistema Fusion registre os diferentes tipos de espaços disponíveis para o atendimento de pacientes, como enfermarias, quartos privativos ou setores específicos. Este cadastro também suporta integrações com outras plataformas hospitalares.

Categorias de Procedimento

O módulo de Categorias de Procedimento organiza os tipos de procedimentos médicos e assistenciais dentro do sistema Fusion. Essa categorização é fundamental para gestão clínica, faturamento e interoperabilidade com outros sistemas.

Países

O módulo de Países permite registrar e manter atualizada a lista de países utilizados em formulários, cadastros de pacientes, integrações e outros contextos do sistema Fusion. Além do nome, é possível configurar siglas, nacionalidades e mapeamentos com sistemas externos.

Estados

O módulo de Estados do Fusion permite o gerenciamento das unidades federativas associadas a cada país previamente cadastrado. Ele é essencial para garantir a consistência em cadastros de pacientes, endereços e integrações.

Cidades

O módulo de Cidades do Fusion permite registrar os municípios vinculados aos estados previamente cadastrados. Essa base é essencial para preenchimento de endereços, localização de unidades e envio de dados padronizados a sistemas integrados.

Bairros

O módulo de Bairros permite organizar a divisão territorial das cidades cadastradas no Fusion. Esse recurso é importante para garantir maior precisão nos registros de endereço e para integração com sistemas externos que utilizam essa estrutura.

Caráter de Atendimento

O módulo Caráter de Atendimento permite classificar a natureza de um atendimento dentro do Fusion. Essa classificação é essencial para o correto registro clínico, faturamento e integração com sistemas externos que exigem esse tipo de informação.

Classificação de Ocupação (C.B.O.)

O módulo de Classificação Brasileira de Ocupações (C.B.O.) permite o cadastro e a manutenção das funções profissionais de saúde utilizadas no sistema Fusion. Essas classificações são importantes para fins de registros profissionais, relatórios, compatibilidade com sistemas externos e integrações com o SUS.

Tipo de Profissional

O módulo Tipo de Profissional permite o cadastro e a gestão de categorias de atuação dos profissionais de saúde no sistema Fusion. Essa informação é essencial para classificações internas, autorizações, permissões e integrações.

Conselho Profissional

O módulo Conselho Profissional permite o cadastro, edição e gestão dos conselhos aos quais os profissionais de saúde estão vinculados, como CRM, COREN, CRO, entre outros. Essas informações são fundamentais para validação profissional, integrações com sistemas externos e processos regulatórios internos.

Duração

O módulo Duração permite o cadastro de períodos de tempo padronizados no sistema, utilizados em prescrições, registros de eventos e integração com outros sistemas. É possível definir unidades como dias, semanas ou meses, bem como associar fatores de conversão e códigos externos.

Especialidade

O módulo Especialidade permite o cadastro de áreas específicas de atuação dos profissionais de saúde, como Anestesiologista, Cardiologista, entre outros. Essas especialidades são vinculadas aos tipos de profissional e utilizadas em diversos pontos do sistema, como autorizações, agendamentos e integrações.

Tipo de Equipe

O módulo Tipo de Equipe permite cadastrar e gerenciar os diferentes tipos de equipes de saúde presentes nas instituições, como Equipe de Saúde da Família ou Equipe de Saúde Bucal. Essas classificações são importantes para organização interna, integrações com sistemas externos e relatórios.

Tipo de Instituição

O módulo Tipo de Instituição permite cadastrar as diferentes naturezas institucionais das unidades de saúde integradas ao sistema Fusion, como clínicas, centros de especialidade, unidades públicas, privadas ou filantrópicas. Esse cadastro é essencial para categorização e integração com sistemas externos.

Tipo de Endereço

O módulo Tipo de Endereço permite cadastrar e classificar os diferentes tipos de endereços utilizados no sistema, como residencial, principal ou padrão. Essa categorização auxilia na organização de dados de contato e é essencial para integrações com sistemas externos.

Tipo de Atendimento

O módulo Tipo de Atendimento permite o cadastro e controle dos diferentes tipos de atendimento realizados nas instituições de saúde, como consultas, emergências, internações e pronto-socorro. Essa categorização é essencial para registros clínicos, relatórios e integração com outros sistemas.

Tipo de Substância

O módulo Tipo de Substância permite o cadastro das classificações de substâncias utilizadas em prescrições, registros farmacológicos e integrações clínicas. Essas informações são essenciais para garantir padronização, rastreabilidade e interoperabilidade com outros sistemas de saúde.

Via Administração

O módulo Via Administração permite cadastrar as diferentes formas pelas quais medicamentos ou substâncias são administrados aos pacientes, como via oral, endovenosa, tópica, entre outras. Esse cadastro é essencial para garantir precisão nas prescrições e compatibilidade com sistemas integrados.

Status de Diagnóstico

O módulo Status de Diagnóstico permite classificar o estágio ou situação de um diagnóstico dentro do prontuário do paciente. Ele auxilia no acompanhamento clínico, na geração de relatórios e na interoperabilidade com outros sistemas.

Status de Atendimento

O módulo Status de Atendimento permite o controle e a classificação do andamento do atendimento de um paciente, como “Aguardando”, “Em Atendimento” ou “Atendimento Realizado”. Esses status são fundamentais para o acompanhamento em tempo real e relatórios gerenciais.

Status de Alergia

O módulo Status de Alergia permite o controle do estado de uma alergia registrada no prontuário do paciente, como "Ativo" ou "Inativo". Essa classificação é fundamental para que a equipe clínica tome decisões seguras e rápidas durante o atendimento.

Status de Agendamento

O módulo Status de Agendamento permite a criação e o gerenciamento dos diferentes estados possíveis de um agendamento no sistema, como por exemplo “Agendado”, “Cancelado”, “Concluído” etc. Esses status ajudam no controle e rastreamento do ciclo completo dos atendimentos agendados.

Status de Solicitação de Procedimento

O módulo Status de Solicitação de Procedimento permite controlar a situação de uma solicitação médica ou assistencial registrada no sistema, como “Solicitado”, “Agendado”, “Cancelado”, entre outros. Essa funcionalidade garante rastreabilidade e organização dos fluxos de atendimento.

Anexos

O módulo Anexos permite cadastrar os tipos de arquivos aceitos e utilizados no sistema, como PDFs, imagens, entre outros. Esse cadastro é essencial para garantir que os documentos recebidos ou exportados sigam um padrão compatível com o ambiente da instituição e com sistemas integrados.

Tipo de Aleitamento Materno

O módulo Tipo de Aleitamento Materno permite o cadastro de classificações referentes às formas de amamentação, como exclusivo, predominante, misto, entre outras. Essa informação é essencial para acompanhamento nutricional e epidemiológico, especialmente em programas de atenção à saúde infantil.

Alergia

O módulo Alergia permite o cadastro de substâncias, medicamentos, alimentos ou outros agentes que possam desencadear reações alérgicas em pacientes. Essas informações são essenciais para garantir a segurança clínica e evitar intercorrências durante atendimentos e prescrições.

Substância

O módulo Substância permite o cadastro das substâncias ativas utilizadas em prescrições, registros de medicamentos, alergias e demais componentes do prontuário clínico. Esse cadastro é essencial para garantir consistência farmacológica e permitir integrações com sistemas externos.

Gêneros

O módulo Gêneros permite cadastrar e gerenciar as categorias de identificação de gênero utilizadas nos cadastros de pacientes e profissionais. Essas informações são fundamentais para registros clínicos, epidemiológicos e operacionais.

Unidades de Medida

O módulo Unidades de Medida permite o cadastro das unidades utilizadas para registros clínicos, prescrições e integrações com sistemas externos. Isso inclui medidas de massa, volume, temperatura, frequência, entre outras.

Frequência

O módulo Frequência permite o cadastro das periodicidades utilizadas em prescrições e tratamentos, como horários fixos, intervalos, doses únicas ou administrações contínuas. Essas definições são fundamentais para garantir precisão nos planos terapêuticos.

Local

O módulo Local permite o cadastro dos ambientes, setores ou áreas físicas da instituição onde ocorrem procedimentos, atendimentos, internações ou alocações específicas. É essencial para a rastreabilidade e organização dos processos assistenciais.

Leitos

O módulo Leitos permite o cadastro e a gestão dos leitos hospitalares no sistema Fusion. Essa funcionalidade é crucial para o controle de disponibilidade, alocação de pacientes e para integrações com outros sistemas hospitalares, garantindo que as informações sobre os leitos estejam sempre atualizadas e corretas.

Medicamento

O módulo Medicamento é utilizado para o cadastro e a gestão de todos os medicamentos utilizados pela instituição no sistema Fusion. Este cadastro é fundamental para a prescrição eletrônica, controle de estoque (quando integrado a sistemas de farmácia), faturamento e para garantir a correta identificação dos medicamentos em integrações com outros sistemas de saúde.

Tabelas

O módulo Tabelas no sistema Fusion é uma área de cadastro genérica e flexível, projetada para gerenciar diversas tabelas de domínio e parametrização utilizadas em diferentes partes do sistema e em integrações. Exemplos incluem tabelas de Classificação Internacional de Doenças (CID), tipos de procedimento (TUSS), motivos de alta, tipos de equipe, entre outras. Manter essas tabelas atualizadas e corretamente mapeadas é crucial para a consistência dos dados e o bom funcionamento das integrações.

Modelo Assistencial

O módulo Modelo Assistencial no sistema Fusion permite o cadastro e a gestão dos diferentes modelos de assistência ou planos de cuidado que podem ser aplicados aos pacientes. Este cadastro é importante para classificar tipos de atendimento, definir protocolos e facilitar a integração com outros sistemas que utilizam essa categorização.

Motivo de Óbito

O módulo Motivo de Óbito no sistema Fusion é dedicado ao cadastro e gerenciamento das causas de óbito. Este cadastro é essencial para registros oficiais, estatísticas vitais, e para a correta documentação clínica e legal. A padronização e o mapeamento para integrações garantem a consistência da informação entre diferentes sistemas.

Motivo de Alta

O módulo Motivo de Alta no sistema Fusion é utilizado para cadastrar e gerenciar os diversos motivos pelos quais um paciente pode receber alta hospitalar ou de um tratamento. Este cadastro é fundamental para registros clínicos, faturamento, estatísticas de saúde e para integrações com outros sistemas, garantindo que a razão da alta seja documentada de forma padronizada.

Severidade de Alergia

O módulo Severidade de Alergia no sistema Fusion destina-se ao cadastro e gerenciamento dos diferentes níveis de severidade que uma reação alérgica pode apresentar. Este cadastro é crucial para a correta classificação das alergias dos pacientes, auxiliando profissionais de saúde na tomada de decisões clínicas e garantindo a segurança do paciente. A padronização e o mapeamento para integrações asseguram a consistência da informação entre sistemas.

Prioridade de Solicitações

O módulo Prioridade de Solicitações no sistema Fusion é utilizado para cadastrar e gerenciar os diferentes níveis de prioridade que podem ser atribuídos a diversas solicitações dentro do ambiente hospitalar, como exames, procedimentos, interconsultas, entre outros. Este cadastro ajuda a organizar o fluxo de trabalho, otimizar o atendimento e garantir que as solicitações mais urgentes sejam tratadas adequadamente. A padronização e o mapeamento para integrações são importantes para a consistência da informação entre sistemas.

Equipes

O módulo Equipes no sistema Fusion é destinado ao cadastro e gerenciamento das diversas equipes de saúde que atuam na instituição. Este cadastro é importante para a organização dos serviços, atribuição de responsabilidades, agendamentos e para integrações com outros sistemas que necessitam identificar as equipes envolvidas no cuidado ao paciente.

Tipos de Detalhes de Recursos

O módulo Tipos de Detalhes de Recursos no sistema Fusion serve para cadastrar e gerenciar categorias ou tipos de informações específicas que podem ser associadas a um "Recurso" (módulo subsequente). Por exemplo, um recurso "Sala Cirúrgica" pode ter um detalhe do tipo "Anestesia" associado a ele. Este cadastro permite uma parametrização flexível e detalhada dos recursos da instituição e é fundamental para o mapeamento correto em integrações.

Detalhes de Recursos

O módulo Detalhes de Recursos é utilizado para cadastrar os valores ou opções específicas para cada categoria definida em Tipos de Detalhes de Recursos. Por exemplo, para o tipo "Anestesia", os detalhes poderiam ser "Com Sedação" e "Sem Sedação". Este cadastro permite criar um sistema de características e valores que podem ser associados aos recursos principais, oferecendo grande flexibilidade na parametrização do sistema.

Recursos

O módulo Recursos é um cadastro central no Fusion, utilizado para definir itens, serviços ou locais que podem ser agendados ou solicitados, como exames, procedimentos, equipamentos ou salas. Este módulo integra as informações cadastradas em "Tipos de Detalhes de Recursos" and "Detalhes de Recursos" para caracterizar completamente cada recurso, além de possuir seus próprios mapeamentos para integrações.

Localidades

O módulo Localidades é utilizado para cadastrar diferentes locais ou regiões, que podem ser tanto áreas físicas do hospital quanto locais anatômicos (ex: "Partes Inferiores"). Este cadastro é utilizado em outras partes do sistema, como no registro de procedimentos ou diagnósticos, para especificar onde uma ação ocorreu ou a que área um achado se refere.

Lateralidades

O módulo Lateralidades é utilizado para cadastrar os conceitos de lado (ex: Esquerdo, Direito, Bilateral), que são essenciais para especificar a localização de procedimentos, diagnósticos ou achados clínicos em órgãos ou membros pares. Este cadastro complementa o módulo "Localidades", garantindo a precisão da informação clínica registrada.

Procedimento

O módulo Procedimento é um dos cadastros centrais do sistema Fusion, utilizado para registrar todos os procedimentos, exames e serviços realizados pela instituição. Isso pode incluir desde exames de imagem, como "RADIOGRAFIA DE TORAX", a exames laboratoriais, como "GLICOSE", e atendimentos, como "ATENDIMENTO DE URGENCIA". Este cadastro é a base para solicitações, agendamentos, registros clínicos e faturamento.

Motivo de Atendimento

O módulo Motivo de Atendimento é utilizado para cadastrar as diversas razões ou tipos de visitas que levam um paciente a procurar atendimento na instituição. Exemplos comuns incluem "Primeira Consulta", "Retorno", "Urgência/Emergência", "Acompanhamento". Este cadastro é importante para fins de agendamento, estatísticas, faturamento e para a correta classificação do encontro do paciente no sistema.

Cadastro Instituição

O módulo Cadastro Instituição é utilizado para registrar outras instituições de saúde, como hospitais, clínicas, laboratórios e unidades de saúde, com as quais a sua organização interage. Este cadastro é essencial para o gerenciamento de transferências de pacientes, recebimento de histórico clínico externo, encaminhamentos e outras funcionalidades que dependem da interoperabilidade entre diferentes estabelecimentos de saúde.

Tipos de Diagnóstico

O módulo Tipos de Diagnóstico é utilizado para cadastrar as diferentes classificações que um diagnóstico pode receber ao longo do atendimento do paciente. Exemplos comuns incluem "Principal", "Inicial", "Final", "Definitivo" e "Preliminar". Este cadastro permite estruturar a informação do diagnóstico, o que é fundamental para a continuidade do cuidado, registros clínicos e para a correta comunicação com outros sistemas de saúde.

Sinais Vitais

O módulo Sinais Vitais no sistema Fusion é utilizado para cadastrar todos os tipos de sinais vitais e medidas antropométricas que podem ser registrados para um paciente. Exemplos incluem "Pressão Sistólica", "Frequência Cardíaca", "Temperatura", "Peso" e "Altura". Este cadastro é fundamental para a criação de formulários de evolução, monitoramento de pacientes e para garantir que os dados coletados sejam padronizados e corretamente integrados com outros sistemas.

Vacinas

O módulo Vacinas no sistema Fusion é utilizado para cadastrar todas as vacinas disponíveis para administração e registro no histórico do paciente. Este cadastro é essencial para o controle do calendário vacinal, campanhas de imunização e para a correta integração com sistemas de saúde governamentais, como o RNDS (Rede Nacional de Dados em Saúde).

Profissionais

O módulo Profissionais é utilizado para o cadastro e gerenciamento de todos os profissionais de saúde que atuam na instituição ou que são referenciados em atendimentos. Manter um cadastro completo e preciso é fundamental para agendamentos, atribuição de responsabilidades, registros de autoria em prontuários, faturamento e integrações com outros sistemas.

Organizações

O módulo Organizações é um cadastro genérico utilizado para registrar diversas entidades que se relacionam com a instituição de saúde, como empresas conveniadas, fornecedores, ou outras organizações parceiras. Este cadastro permite centralizar informações sobre essas entidades para uso em diferentes contextos dentro do sistema Fusion.

Raça/Cor

O módulo Raça/Cor é utilizado para cadastrar as categorias de raça/cor utilizadas no registro demográfico dos pacientes. Este cadastro é fundamental para a coleta de dados de saúde, para atender a requisitos legais e para a integração com sistemas que exigem essa informação, geralmente seguindo o padrão do IBGE (Branca, Preta, Parda, Amarela e Indígena).

Etnia

O módulo Etnia é utilizado para cadastrar as etnias dos pacientes, complementando a informação de "Raça/Cor". Este cadastro é particularmente importante para o registro de populações específicas, como povos indígenas e comunidades quilombolas, sendo um dado crucial para políticas de saúde pública, estudos epidemiológicos e para a adequação do cuidado às especificidades culturais.

Mantenedoras

O módulo Mantenedoras é utilizado para cadastrar as entidades mantenedoras das instituições de saúde. Uma mantenedora é a organização-mãe ou a entidade legal responsável por uma ou mais instituições, como uma fundação, uma secretaria de saúde ou um grupo empresarial. Este cadastro é importante para a estrutura organizacional e para fins de relatórios e integrações em um nível corporativo.

Tipo de Gestão de Mantenedoras

O módulo Tipo de Gestão de Mantenedoras é utilizado para cadastrar as diferentes classificações ou naturezas jurídicas das entidades mantenedoras, como "Pública", "Privada", "Filantrópica", etc. Este cadastro serve para categorizar as mantenedoras registradas no módulo anterior, facilitando a geração de relatórios, filtros e a aplicação de regras específicas para cada tipo de gestão.
